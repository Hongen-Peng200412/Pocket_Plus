Step 4: 网络框架

本文的新版 Stage1 网络以单个 BOX 为基本处理单元，同时接收规则体素表示和与该 BOX 对应的原子集合。与旧版相比，新版模型不再是“体素分支 + 点云分支 + 末端分类头”的简单串联，而是在前端加入了嵌入头 embed head，在点云主干内部加入了伪原子节点，并通过多层 voxel-to-point 融合与有限轮 recycle 机制实现跨模态迭代细化。整体上，网络仍以 atom-level 二分类作为最终目标，但其信息流已经扩展为“原子预编码 -> 体素/点并行建模 -> 多层融合 -> atom head 判别”的联合框架。

模块：

1.嵌入头 embed head

首先，embed head 以当前 BOX 内原子及其邻域 buffer 原子为输入，对 49 维原子化学特征进行线性投影后，再按照 z-order、转置 z-order、Hilbert 和转置 Hilbert 等多种序列化顺序送入若干层点注意力 block。该模块本身并不直接产生最终 pocket 预测，而是先在点域内对稀疏、离散、维度较高的原子信息进行压缩编码，为后续体素分支和点云分支同时提供结构先验。

新版 embed head 的关键不只是“多加一个点网络”，而是其渐进式裁剪机制。具体而言，输入原子并不限制为 BOX 内部原子，而是先保留一定 buffer 范围内的外围原子；随后在共享 trunk、体素专用分支和点专用分支的每一轮 block 之后，都按照逐步收缩的缓冲半径剔除距离 core box 过远的原子，最终保留下来的恰好是当前 BOX 所需的核心原子集合。与一次性硬裁剪相比，这种逐层收缩的方式既保留了外围原子的上下文信息，又避免了大量无关原子长期参与后续计算，从而兼顾感受野、显存开销和训练稳定性。

在输出端，embed head 分成两条路径：一条体素路径将 per-atom hidden feature 按原子所在的连续体素坐标 scatter 到规则网格上，得到低通道的 voxel embedding，并与原始密度图及差图通道拼接后送入体素 backbone；另一条点路径输出紧凑的 per-atom 特征，并与原始 atom feature 经过线性投影后的结果做残差融合，作为点云 backbone 的输入。这样做的动机在于：若将 49 维稀疏原子特征直接整体体素化并并入 3D 卷积分支，会显著增加显存负担；而 embed head 可以先在点域完成压缩，再把信息密度更高的结构表示注入体素分支。

2.体素分支

体素分支基于 Recycling Attention U-Net (RAUNet) 构建，整体保持四级编码器-解码器结构。网络首先将当前轮的 BOX 体素输入与上一轮 recycle 得到的体素隐状态融合，然后通过多次下采样逐步扩大感受野，并在最深层瓶颈处引入带三维旋转位置编码的自注意力模块，以增强模型对长程空间依赖和全局密度上下文的建模能力。

在解码阶段，模型通过逐级上采样和跳跃连接恢复空间分辨率，同时使用 attention gate 对来自编码端的特征进行筛选，从而抑制背景区域中的无关响应。为兼顾不同尺度下的 pocket 线索，最终高分辨率体素特征并非来自单一卷积核，而是将 3×3、5×5 和 7×7 三个卷积分支的结果拼接后再映射得到。除最终体素特征外，体素分支还显式导出多个中间尺度的命名特征，以供后续 voxel-to-point 融合使用；同时保留一个轻量级体素辅助头，用于维持体素语义表征的稳定性。

3.点云分支

点云分支以 BOX 内原子为主体，并在需要时与伪原子共同组成扩展点集。对于每个原子，模型使用以 BOX 中心为原点的居中三维坐标，以及由 embed head 输出或原始特征映射得到的原子表示，构造 Point Transformer V3 主干的输入。与把上一轮点特征简单旁路相加不同，新版模型会先将 recycle 特征归一化并投影回 atom feature 空间，再与当前原子特征相加，随后重新做输入嵌入，从而使 recycle 信息真正参与下一轮点表示的重新编码。

在主干结构上，本文沿用 PTV3 的 encoder-decoder 设计，并使用 z-order、转置 z-order、Hilbert 和转置 Hilbert 四种空间填充曲线对三维点集进行序列化，将无序点云转化为更适合局部注意力建模的一维序列。当前实现的点云内部离散化粒度为 0.25 Å，embedding 与条件位置编码均采用 pointconv 风格的邻域聚合，以同时保留局部几何结构和较大尺度的上下文关系。更重要的是，点分支在输入嵌入、编码器各层和最终输出层均保留命名中间变量，这一设计并非仅用于可视化，而是为体素到点的信息融合提供明确的挂接位置。

4.分类头 atom head

在得到融合后的点特征之后，模型并不立即使用单层线性分类器输出结果，而是构造 atom token 并交给独立的 atom head 进一步判别。具体而言，模型将最终点特征、原子的居中世界坐标以及 atom valid mask 拼接为 token，经线性投影统一到隐藏空间后，再送入多层序列化注意力堆叠进行上下文建模。

该分类头与点云主干共享“序列化 + 局部注意力”的思想，但其职责更聚焦于 atom-level 判别本身：一方面利用 BOX 内原子之间的局部和中程依赖关系对最终表示进行再组织，另一方面又保持预测输出仍然严格对应到真实原子。经过多层 attention 和 MLP 映射后，模型直接输出每个原子的二分类 logit，从而避免了“先输出体素概率，再后处理映射回原子”所带来的语义偏移和精度损失。

机制：

1.伪原子机制

为增强点分支对 BOX 内空腔区域、局部密度上下文以及结构间隙的表达能力，本文引入了伪原子机制。伪原子并非来自真实结构注释，而是在每个 BOX 的前向过程中根据真实原子数自适应生成的一组额外消息节点。其空间位置在 BOX 范围内采样后，还会经过“过近删除”“过远删除”和聚类稀疏化等步骤，以避免伪原子过度堆积或远离真实结构主体。

在当前实现中，伪原子只在点云 backbone 阶段参与消息传递，而不进入 embed head，也不参与最终 atom head 的监督输出。它们与真实原子按交错方式注入混合点集，在点分支中与真实原子共同接受注意力建模；完成点分支编码后，再被显式移除，只保留真实原子的特征用于最终分类。因此，伪原子的作用更接近“辅助上下文节点”，而不是“带标签的额外样本”。

2.信息融合机制

新版模型的跨模态融合采用单向 voxel-to-point 设计，即以体素分支提供连续空间密度上下文，以点分支承担原子级几何与化学判别。具体地，模型并不是在点云主干结束后只融合一次体素特征，而是在多个命名层位上即时执行融合：先由原子的居中世界坐标恢复其在 BOX 内的连续体素坐标，再在对应尺度的体素特征图上用三线性插值采样，得到与当前点一一对应的 voxel feature；随后将采样特征与当前点特征拼接，并通过线性投影映射回点特征空间。

这种多层融合使点分支在编码中后期不断接收来自密度图的局部纹理、残差信号和空间上下文，而不仅在末端被动读取一份静态体素表示。当前实现将融合层位布置在点云编码器的中高层以及最终输出层，对应地从体素分支读取多个尺度的中间特征和最终高分辨率特征，从而同时兼顾粗尺度上下文与细尺度局部结构。

3.recycle 机制

除上述模块与融合方式外，本文还在 Stage1 中引入有限轮 recycle 机制，对同一 BOX 内的体素表示和点表示进行迭代细化。与 AlphaFold2 中的 recycle 思想类似，模型在一次前向过程中可以运行多轮“体素建模-点云建模-atom 判别”循环：体素分支将上一轮的 voxel hidden state 作为额外输入再次编码，点分支则将上一轮的 point feature 投影回 atom feature 空间后重新编码。

需要指出的是，embed head 位于 recycle 循环之外，只在前向开始时执行一次；而 recycle 仅作用于体素分支和点分支之间的主干表征。训练阶段，模型可在 1 到 3 轮之间随机采样 recycle 次数，以增强模型对不同迭代深度的鲁棒性；推理阶段则使用完整轮数。为了控制跨轮梯度传播带来的不稳定性，实现中还对 recycle 状态进行了显式 detach。有限轮的迭代精化使体素上下文与原子表示能够逐步对齐，从而提高最终 atom-level pocket 判别的稳定性和精度。

综上，新版模型并非简单地把体素网络与点网络串联，而是通过 embed head、体素分支、点云分支和 atom head 四个模块，以及伪原子、多层信息融合和 recycle 三个机制，共同构成一个面向 atom-level pocket 识别的联合建模系统。
