第一大步的数据实现细则
=========================================================================================================================

一、样本解析
----------------------------------------------
样本名形如：
`{pdb_id}_{instance_id}_{Rxx}_{Ryy}_{Rzz}[_C]`
`BoxPointDataset` 必须自己提供函数解析出：
1. `pdb_id`
2. `instance_id`
3. `is_center_box`
建议从右向左解析，不要写脆弱的 split 逻辑。


二、BOX 数据读取
----------------------------------------------
继续使用现有三个 BOX 文件夹：
1. `emdb_BOX`
2. `pdb_feature_BOX`
3. `pdb_label_BOX`
读取原则：
1. 特征拼接逻辑直接继承旧 dataset 思路。
2. `emdb` 通道继续做 z-score。
3. `hardmask` 继续按照旧逻辑从非 emdb 通道是否全零生成。
4. 现在读取时可以把 `origin / voxel_size / x_range / y_range / z_range` 一并读出来。
5. `x_range / y_range / z_range` 只作为几何元信息保留，可以暂时不参与监督区域构造。


三、原子读取
----------------------------------------------
从原始样本目录读取：
1. `atoms.npz`
2. `labels.npz`
首版直接使用：
1. `atoms.npz["coords"]`
2. `atoms.npz["features"]`
3. `labels.npz["binding_mask"]`
4. `labels.npz["pocket_class_ids"]`
5. `labels.npz["instance_ids"]`
其中：
1. 第一大步主标签用 `binding_mask`。
2. `pocket_class_ids` 暂时只保留，不作为主训练标签。


四、BOX 内原子筛选
----------------------------------------------
规则固定如下：
1. 先取 core box 内原子。
2. 再取 BOX 外扩 `atom_buffer_radius` 后范围内的 buffer 原子。
3. 首版训练默认只监督真正落在有效监督区域内的 core 原子；buffer 原子主要用于提供上下文，一律不参与监督。
4. `atom_is_in_core_box` 保留为几何/调试字段；真正用于训练监督的是 `atom_valid_mask`。
第一大步不要把筛选逻辑写复杂。
不要做：
1. whole-PDB point branch
2. 复杂 pseudo atom 采样
3. 复杂候选点系统
4. 不需要考虑因为原子数目过多而裁剪原子


五、atom_feat 的首版构成
----------------------------------------------
第一大步里，`atom_feat` 建议固定为：
原始 49 维 atom features
即：
`F_atom = 49`
`core_flag` 不再拼进 `atom_feat`。
`atom_valid_mask` 也不塞进 `atom_feat`，而是在构造 atom token 时作为单独 1 维监督表征拼接，避免把监督语义混进底层原子原始特征。
相对 BOX 中心的几何信息也不再重复塞进 `atom_feat`，而是统一通过 `atom_coord_centered_world` 单独提供给 point branch / point head。
同时也要允许 F_atom 可以在后续代码中变化。


六、坐标系统
----------------------------------------------
这是第一大步最容易出错的地方，必须统一。
固定定义如下：
1. `atom_coord_world`
   - 世界坐标
   - `(x, y, z)`
2. `atom_coord_local_voxel`
   - 连续 voxel 坐标
   - `(x, y, z)`
   - 定义为：
     `atom_coord_local_voxel = (atom_coord_world - box_origin_world) / voxel_size_world`
   - 这里固定采用“角点语义”，不是“voxel 中心语义”
   - 若 atom 恰好落在 `(ix, iy, iz)` 这个 voxel 的中心，则
     `atom_coord_local_voxel = (ix + 0.5, iy + 0.5, iz + 0.5)`
3. `atom_coord_centered_world`
   - 相对 BOX 中心的世界坐标
   - `(x, y, z)`
   - 供 point branch / point head 使用
注意：
1. `grid` 的空间轴是 `(Z, Y, X)`。
2. `origin / voxel_size / atom_coord_*` 的坐标顺序是 `(x, y, z)`。
3. 三线性插值时必须显式处理这两套顺序的映射。
4. `_voxel_xyz_to_grid_sample_xyz()` 与 `grid_sample()` 统一按 `align_corners=True` 的约定实现，不要混用别的归一化规则。
5. 由于 `atom_coord_local_voxel` 采用角点语义，送入 `grid_sample()` 前必须先减去 `0.5`，把它转成“voxel 中心 index 语义”：
   - `x_center = x - 0.5`
   - `y_center = y - 0.5`
   - `z_center = z - 0.5`
6. 在此基础上，归一化公式固定为：
   - `grid_x = 2 * x_center / (W - 1) - 1`
   - `grid_y = 2 * y_center / (H - 1) - 1`
   - `grid_z = 2 * z_center / (D - 1) - 1`


七、数据增强
----------------------------------------------
第一大步里训练集继续做 90 度随机旋转，但不能直接偷用旧逻辑。
原因：
旧 `random_rotation90_plus()` 只会旋转数组最后三维，它不会更新 point 坐标数值，如果继续直接调用旧函数，voxel 与 atom 会立刻错位。
因此第一大步必须自己实现同步旋转：
1. 先随机采样 `axis1 / axis2 / k`
2. 对：
   - `voxel_grid`
   - `voxel_label`
   - `hardmask`
   用 `np.rot90`
3. 对：
   - `atom_coord_local_voxel`
   - `atom_coord_centered_world`
   做同一套几何变换
4. 其中 `atom_coord_local_voxel` 保留角点语义，因此与 `np.rot90` 对齐的坐标公式固定为：
   - `new_axis1 = old_size_axis2 - old_axis2`
   - `new_axis2 = old_axis1`
   这里不能写成 `old_size_axis2 - 1 - old_axis2`，否则会稳定少 `1.0` voxel。
5. `atom_coord_centered_world` 仍然按以 BOX 中心为原点的世界坐标来旋转，公式可保持：
   - `new_axis1 = - old_axis2`
   - `new_axis2 = old_axis1`
6. 首版默认 `voxel_size_world` 三轴相同或近似相同；若不满足，直接报错，不在第一版支持各向异性体素下的 90 度轴交换旋转。
这部分一定写成独立函数，不要散落在 `__getitem__()` 里。

阶段 A：先做 dataset

2. batch 能正确 concat atom
3. `atom_batch_index / atom_counts / atom_offsets` 正确

7. recycle 接口已经纳入 C 阶段，至少包括 `enable_recycling / max_recycles / randomize_recycles / detach_recycle_states`

注意事项
=========================================================================================================================
第二大步的预留接口
----------------------------------------------
虽然本文件重点是第一大步，但第一大步实现时必须给第二大步留好口子。
要求如下：
1. `Stage1PointBackbone.forward()` 的输入输出接口在第一大步就定下来。
2. 第二大步优先在完整依赖环境中训练、消融、调参；如需替换点分支内部主干，也尽量不改上层接口。
3. `VolumePointStage1Model` 不要把某一种 point backend 或某一种固定融合位置写死在 `forward()` 里。
4. `wrapper` 不要把损失写死为“只有 atom loss”。
5. `dataset` 一开始就返回完整 point 数据，第二大步不再重写数据入口。
6. `Stage1PointBackbone` 内部继续保持接近 PTV3 的标准 point dict / point state 组织方式，第二大步优先保持上层数据组织不变。
7. voxel->point 采样和 point 侧融合要从第一大步开始抽象出清晰接口，后续允许继续扩展 sampler / fusion / recycle 策略，但第一版只落地当前最小实现。


第一大步最容易做坏的地方
----------------------------------------------
一、坐标顺序混乱
1. 世界坐标是 `(x, y, z)`
2. voxel grid 空间轴是 `(Z, Y, X)`
3. 如果不显式处理，三线性插值很容易采到错位位置

二、旋转增强只转 voxel 不转 atom
这是最危险的实现错误之一。
如果发生，整个方案虽然能跑，但学到的是错的。

三、过早引入太多新变量
第一大步不要同时做：
1. 完整 PTV3
2. 伪原子
3. 多种融合策略
4. point -> voxel 回写
5. whole-PDB point branch
