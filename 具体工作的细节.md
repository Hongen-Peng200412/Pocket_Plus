# 当前主线

当前最新版主训练入口固定为：

1. `sbatch/sparse_h100.sbatch`
2. `python Pocket_Plus/src/train.py --config=sparse_h100`
3. `configs/sparse_h100.yaml`
4. `model: sparse`
5. `dataset: fused`
6. `train: default_h100`

其中：

- `sparse_h100` 是当前主线，不再把它视为“未来要接上的版本”。
- `dense` 是更密集的融合配置（四融合点），用于对照或消融。
- `sparse_embed` 在 `sparse` 基础上加入 `Stage1EmbedHead` 前置模块，用在线原子编码替代离线 pdb_feature_BOX。
- `zeros` 是点分支/融合的调试后端，不是主训练配置。

当前已有的辅助顶层入口还包括：

- `unfused_h100`：在 `sparse_h100` 基础上，将数据集换成 `unfused`（去掉 pdb_feature_BOX，只读 emdb_BOX + pdb_label_BOX）。
- `dense_h200`：在 `sparse_h100` 基础上，将模型换成 `dense`，训练配置换成 `default_h200`。
- `dense_unfused_h200`：在 `dense_h200` 基础上，将数据集换成 `unfused`。

`sbatch/sparse_h100.sbatch` 会在运行时继续覆盖：

- `train.nnodes=$SLURM_JOB_NUM_NODES`
- `train.devices=$GPUS_PER_NODE`

因此文档中所有 batch 相关描述都必须区分：

- 配置文件里的“每卡 batch size”
- 由 Slurm 决定的 world size
- 梯度累积后的有效 batch

# 数据子系统

## 1. 数据集定位

当前主数据集是 `src/datasets/box_point_dataset.py` 中的 `BoxPointDataset`。

它负责一次性返回 stage1 训练所需的完整样本字典，包含：

- voxel 输入
- voxel 标签
- voxel 辅助监督掩码
- atom 坐标
- atom 特征
- atom 标签
- atom 监督掩码
- 样本级元信息



## 2. 当前 `fused` 配置

`configs/dataset/fused.yaml` 当前主配置为：

- `_target_: src.datasets.box_point_dataset.BoxPointDataset`
- `data_folder_names = ["emdb_BOX", "pdb_feature_BOX", "pdb_label_BOX"]`
- `class_folder_names = ["metal_ion", "peptide", "nucleic", "small_molecule"]`
- `class_mapping = [0, 1, 0, 0, 1]`
- `atom_buffer_radius = 0.0`
- `valid_crop_margin = 0`
- `enable_random_rotation = true`

`class_mapping = [0, 1, 0, 0, 1]` 的当前含义是：

- 原始标签 `0` 映射到 `0`
- 原始标签 `1` 映射到 `1`
- 原始标签 `2` 映射到 `0`
- 原始标签 `3` 映射到 `0`
- 原始标签 `4` 映射到 `1`

因此当前训练虽然读取的是多类 `pocket_class_ids`，但经过映射后是二分类训练。


## 3. 样本名解析

当前样本名格式固定为：

`{pdb_id}_{instance_id}_{rxx}_{ryy}_{rzz}[_C]`

`BoxPointDataset._parse_sample_name()` 当前会解析出：

- `pdb_id`
- `instance_id`
- `rxx`
- `ryy`
- `rzz`
- `is_center_box`

其中 `pdb_id` 会统一转为小写。


## 4. BOX 侧数据读取

`_load_box_npz_triplet()` 的当前行为固定为：

1. 遍历 `data_folder_names`
2. `label` 文件夹读成 `voxel_label`
3. 其它文件夹沿 channel 维拼接成 `voxel_grid`
4. 含 `emdb` 的通道会做整图 z-score
5. 从 npz 中读取或反推：
   - `box_origin_world`
   - `voxel_size_world`
   - `x_range`
   - `y_range`
   - `z_range`
6. 基于非 EMDB 通道是否非零，生成 `hardmask`

当前 `hardmask` 的语义是：

- 只看非 EMDB feature 通道
- 某体素只要有任一非 EMDB feature 非零，则 `hardmask=1`
- 若没有任何非 EMDB feature 通道，则返回全零 `hardmask`

注意：

- `x_range / y_range / z_range` 当前会在内部读取，但不会进入最终 batch 主契约。
- 文档中不要把它们写成 `__getitem__()` 的返回字段。


## 5. 原子侧数据读取与缓存

`_load_structure_npz_cached()` 当前从：

- `atoms.npz`
- `labels.npz`

中读取：

- `coords -> atom_coord_world`
- `features -> atom_feature_raw`
- `binding_mask`
- `pocket_class_ids`
- `instance_ids`

但当前真正进入主训练标签的是：

- `pocket_class_ids`

不是：

- `binding_mask`

`binding_mask` 当前只是随原始标签一起被读取，并未作为主损失标签使用。

当前实现还维护每个 DataLoader worker 内部的轻量 LRU cache：

- key 为 `pdb_id`
- value 为对应结构的原子与标签字典


## 6. 原子筛选规则

`_select_atoms_for_box()` 当前固定按几何规则筛选：

1. 先用当前 BOX 的 `[origin, max)` 定义 `core_mask`
2. 再用 `atom_buffer_radius` 做外扩，得到 `selected_mask`
3. 返回：
   - `selected_idx`
   - `atom_is_in_core_box`

在当前主配置下：

- `atom_buffer_radius = 0.0`

因此现行训练里：

- `selected_mask == core_mask`
- 不存在额外 buffer atom

也就是说，代码支持 buffer 原子，但当前 `sparse_h100` 主线并没有启用这部分上下文。


## 7. 原子标签规则

当前原子标签来自：

1. `labels.npz["pocket_class_ids"]`
2. 再经过 `_apply_class_mapping()`

因此当前 `atom_label` 是：

- `selected_idx` 对应原子的类别 ID
- 经过 `class_mapping` 后的最终训练标签

当前主线不要再写成：

- “原子主标签直接用 `binding_mask`”


## 8. 原子特征规则

当前 `atom_feat` 由 `_build_atom_features()` 生成，第一版保持为：

- 从 `atom_feature_raw[selected_idx]` 直接取出
- dtype 为 `float32`
- 当前主配置下特征维度是 `49`

以下内容当前都没有被拼进底层 `atom_feat`：

- `atom_is_in_core_box`
- `atom_valid_mask`
- 相对 BOX 中心的几何坐标

这些信息的当前处理方式分别是：

- `atom_is_in_core_box` 作为单独字段保留
- `atom_valid_mask` 作为单独字段保留，并在 atom token 阶段再拼接
- 几何坐标通过 `atom_coord_centered_world` 单独提供


## 9. 坐标系统

当前代码中需要严格区分三套坐标：

### 9.1 `atom_coord_world`

- 形状 `(N_selected, 3)`
- 顺序 `(x, y, z)`
- 世界坐标系

### 9.2 `atom_coord_local_voxel`

- 形状 `(N_selected, 3)`
- 顺序仍是 `(x, y, z)`
- 连续 voxel 坐标
- 语义是 corner 语义，不是 center-index 语义

定义为：

`(atom_coord_world - box_origin_world) / voxel_size_world`

若某原子恰好落在体素中心，则该坐标会带 `0.5` 偏移。

### 9.3 `atom_coord_centered_world`

- 形状 `(N_selected, 3)`
- 顺序 `(x, y, z)`
- 以 BOX 中心为原点的世界坐标

当前 point 分支与 voxel->point 采样主链路，使用的是：

- `atom_coord_centered_world`

当前 `stage1_model.py` 在采样前会从：

- `atom_coord_centered_world`
- `voxel_size_world`
- `box_shape_zyx`

重新恢复出局部 voxel 坐标，再送入 `grid_sample()`。

因此：

- `atom_coord_local_voxel` 目前属于数据契约、调试字段和未来扩展接口
- 当前主线采样时并没有直接读取 batch 里的 `atom_coord_local_voxel`


## 10. 监督掩码

### 10.1 `voxel_valid_mask`

由 `_build_voxel_valid_mask()` 生成。

规则是：

- 初始为全 `True`
- 在六个面各裁掉 `valid_crop_margin` 个 voxel

当前主配置：

- `valid_crop_margin = 0`

因此当前主线中：

- `voxel_valid_mask` 全为 `True`

### 10.2 `atom_valid_mask`

由 `_build_atom_valid_mask()` 生成。

当前规则是：

1. 原子必须在 `core box` 内
2. 原子连续 voxel 坐标必须落在裁边后的有效区域内

在当前主配置下：

- `atom_buffer_radius = 0.0`
- `valid_crop_margin = 0`

因此当前主线中：

- `atom_valid_mask == atom_is_in_core_box`
- 又由于当前没有 buffer atom，实际等价于“当前样本中所有被选中原子都参与监督”

必须注意：

- 这是“当前配置结果”
- 不是“代码能力上永远没有 buffer/crop”


## 11. 随机旋转增强

当前训练态支持对 voxel 与 atom 做完全同步的 90 度旋转增强。

同步旋转覆盖的字段包括：

- `voxel_grid`
- `voxel_label`
- `hardmask`
- `voxel_valid_mask`
- `atom_coord_local_voxel`
- `atom_coord_centered_world`
- `atom_coord_world`
- `box_shape_zyx`
- `atom_valid_mask`

当前实现还带有两个约束：

1. `voxel_size_world` 必须近似各向同性
2. 若是奇数次 90 度轴交换，旋转平面的两轴长度必须一致

当前主配置里：

- `enable_random_rotation = true`

仅在训练模式下启用。


## 12. `__getitem__()` 当前返回契约

当前样本字典返回：

- `voxel_grid`: `(C, D, H, W)`, `float32`
- `voxel_label`: `(D, H, W)`, `int64`
- `hardmask`: `(D, H, W)`, `int64`
- `voxel_valid_mask`: `(D, H, W)`, `bool`
- `box_origin_world`: `(3,)`, `float32`
- `voxel_size_world`: `(3,)`, `float32`
- `box_shape_zyx`: `(3,)`, `int64`
- `atom_coord_world`: `(N, 3)`, `float32`
- `atom_coord_local_voxel`: `(N, 3)`, `float32`
- `atom_coord_centered_world`: `(N, 3)`, `float32`
- `atom_feat`: `(N, 49)`, `float32`
- `atom_label`: `(N,)`, `int64`
- `atom_is_in_core_box`: `(N,)`, `bool`
- `atom_valid_mask`: `(N,)`, `bool`
- `sample_name`
- `pdb_id`
- `class_name`
- `instance_id`
- `is_center_box`

- 不会在 `torch.cat([])` 时报错

# 模型子系统

## 0. `Stage1EmbedHead`（可选前置模块）

当前实现位于 `src/model/stage1_embed_head.py`。

它当前负责：

1. 在 recycle 循环外只执行一次
2. 对原子做共享 trunk 编码（若干层 PTV3 `Block`）
3. 分叉输出：
   - 体素分支：经若干层 voxel-specific Block 后，`scatter_to_voxel_grid` → `voxel_pdb_embed_grid`
   - 点分支（可选）：经若干层 point-specific Block 后，输出 `embed_point_feat`
4. 渐进裁剪 buffer 原子：每个 Block 后按 `trunk_buffer_radii` / `voxel_buffer_radii` / `point_buffer_radii` 裁剪
5. 作为过滤器：输出裁剪后的原子字段 + `global_keep_mask`，供 `VolumePointStage1Model` 替换 batch

当前 embed head 输出的字段包括：

- `voxel_pdb_embed_grid`: `(B, embed_voxel_out_channels, D, H, W)`
- `embed_point_feat`: `(sumN', embed_point_out_channels)` 或 `None`
- `atom_feat`: `(sumN', C)` 裁剪后
- `atom_coord_centered_world` / `atom_batch_index` / `atom_offsets` / `atom_coord_local_voxel` / `atom_is_in_core_box`: 裁剪后
- `global_keep_mask`: `(sumN,)` bool

当前 `sparse_embed` 主配置下：

- `embed_voxel_out_channels = 16`
- `embed_point_out_channels = 64`
- `num_trunk_blocks = 3`，`num_voxel_blocks = 2`，`num_point_blocks = 2`
- `trunk_buffer_radii = [8.0, 6.0, 4.0]`
- `voxel_buffer_radii = [3.0, 2.0]`
- `point_buffer_radii = [3.0, 2.0]`

当前 `sparse_h100` 主线不启用 embed head（`embed_head = null`）。


## 1. `Stage1VoxelBackbone`

当前实现位于 `src/model/stage1_voxel_backbone.py`，继承 `SimpleUnet`。

它当前负责：

1. 接收 `voxel_grid`
2. 通过 `set_input_channels()` 支持 lazy 初始化输入通道
3. 返回命名字典形式的 voxel 中间特征
4. 返回 `voxel_logits_aux`
5. 返回 `voxel_recycle_out`

当前稳定的命名 voxel 特征包括：

- `voxel_ds_0`
- `voxel_ds_1`
- `voxel_ds_2`
- `voxel_ds_3`
- `voxel_ds_4`
- `voxel_c4`
- `voxel_c3`
- `voxel_c2`
- `voxel_c1`
- `voxel_c0`
- `voxel_final`

当前 `sparse` 主线只显式请求：

- `voxel_c4`
- `voxel_final`


## 2. `Stage1PointBackbone`

当前实现位于 `src/model/stage1_point_backbone.py`。

当前支持的后端只有两种：

- `backend: ptv3`
- `backend: zeros`

当前主线默认使用：

- `backend: ptv3`

当前点分支的稳定命名变量包括：

- `point_input_feat`
- `point_embed`
- `point_enc0`
- `point_enc1`
- `point_enc2`
- `point_enc3`
- `point_enc4`
- `point_dec3`
- `point_dec2`
- `point_dec1`
- `point_dec0`
- `point_feat`

这些变量名不是草案，而是当前融合 hook 真正依赖的命名契约。

当前点分支输入为：

- `atom_feat`
- `atom_coord_centered_world`
- `atom_batch_index`
- `atom_offsets`
- 可选 `recycle_in`

当前点分支输出为：

- `point_feat`
- `point_state`
- `point_recycle_out`
- `point_feature_dict`

当前点分支的可配参数还包括（均在 `dense.yaml` 的 `point_backbone` 中定义）：

- `embedding_impl`: `"sparseconv"` 或 `"pointconv"`，控制 Embedding 层的实现方式
- `cpe_impl`: `"sparseconv"` 或 `"pointconv"`，控制 Block CPE 的实现方式
- `embedding_receptive_field`: pointconv Embedding 世界坐标感受野（Å）
- `pointconv_embed_max_neighbors`: pointconv Embedding 最大邻居数
- `pointconv_block_max_neighbors`: pointconv Block CPE 最大邻居数
- `enc_cpe_kernel_size` / `dec_cpe_kernel_size`: sparseconv 模式每层 CPE kernel size
- `enc_cpe_receptive_field` / `dec_cpe_receptive_field`: pointconv 模式每层 CPE 感受野
- `act_layer_name`: 统一激活函数类型（与 `dense.yaml` 全局配置一致）
- `ffn_type`: Block FFN 类型，`"mlp"` 或 `"gated"`


## 3. `VolumePointStage1Model`

当前实现位于 `src/model/stage1_model.py`，是 stage1 总装配模块。

当前职责固定为：

1. （可选）调用 embed head（在 recycle 循环外只执行一次）
2. 调用 voxel 分支（当 embed head 启用时，体素输入为 `cat(voxel_grid, voxel_pdb_embed_grid)`）
3. 调用 point 分支
4. 在命名点变量处触发 voxel->point 融合
5. 构造 atom token
6. 经过 `Stage1SerializedAttentionStack` 做 atom head
7. 处理 recycle 循环

### 3.0 embed head 集成

当 `embed_head` 不为 `None` 时，`forward()` 会在 recycle 循环前执行以下操作：

1. 调用 `self.embed_head(...)` 获取裁剪后的原子字段与体素嵌入网格
2. 浅拷贝 batch dict，用裁剪后的字段替换 `atom_feat` / `atom_coord_centered_world` / `atom_batch_index` / `atom_offsets` / `atom_coord_local_voxel` / `atom_is_in_core_box`
3. 通过 `global_keep_mask` 同步裁剪 `atom_label` / `atom_valid_mask` / `atom_coord_world`
4. （可选）残差加和：`batch["atom_feat"] = proj(atom_feat) + gate * embed_point_feat`
   - `self.embed_point_add_proj`: `nn.Linear(atom_feature_dim -> embed_point_out_channels)`
   - `self.embed_point_gate`: `nn.Parameter`，初始化为 0.01

当前 `set_input_channels()` 会自动加上 `embed_voxel_out_channels`：

- 体素 backbone 实际输入通道 = 数据通道 + embed_voxel_out_channels

### 3.0.1 `Stage1SerializedAttentionStack` 架构

atom head 的注意力堆叠当前使用 PTV3 `Block` 作为每层的基本单元，每层包含：

- CPE（条件位置编码）：由 `atom_head_cpe_impl` 控制，默认 `"none"`
- `SerializedAttention`: 序列化多头注意力
- FFN：由 `atom_head_ffn_type` 控制，默认 `"none"`（无 FFN）

当前可配参数包括：

- `atom_head_cpe_impl`: `"none"` / `"sparseconv"` / `"pointconv"`
- `atom_head_ffn_type`: `"none"` / `"mlp"` / `"gated"`
- `atom_head_mlp_ratio`: FFN 隐藏层膨胀倍率（默认 4）
- `atom_head_drop_path`: 随机深度 drop 概率（默认 0.0）
- `atom_head_pre_norm`: 是否使用预归一化（默认 true）
- `atom_head_cpe_kernel_size` / `atom_head_cpe_receptive_field` / `atom_head_pointconv_max_neighbors`

### 3.1 当前支持的采样与融合方式

当前仅支持：

- `sampler_modes = {"trilinear", "nearest"}`
- `point_fusion_modes = {"concat_linear"}`

其它 sampler/fusion 策略仍未落地。

### 3.2 当前 sparse 主线融合点

当前 `sparse` 主线只在两个点变量上做融合：

1. `point_enc4 <- voxel_c4`
2. `point_feat <- voxel_final`

不要再把 `enc2/enc3` 写成 `sparse_h100` 默认行为。

### 3.3 当前 atom token 组成

当前 atom token 明确定义为：

1. `fused_point_feat`
2. `atom_coord_centered_world`
3. `atom_valid_mask` 转成 float 后的一维标记

即：

`atom_tokens = concat(fused_point_feat, atom_coord_centered_world, atom_valid_mask)`

这里拼接的是：

- `atom_valid_mask`

不是：

- `atom_is_in_core_box`

### 3.4 当前 recycle 规则

当前默认启用 recycle，配置来自 `dense.yaml`：

- `enable_recycling: true`
- `max_recycles: 3`
- `randomize_recycles: true`
- `detach_recycle_states: true`

当前行为是：

- 若关闭 recycle，则只跑 1 轮
- 若训练态且开启随机 recycle，则从 `1..max_recycles` 中均匀采样轮数
- 否则固定跑 `max_recycles` 轮


## 4. 当前前向主链路

当前真实前向链路固定为：

1. `BoxPointDataset.__getitem__()` 返回 dict sample
2. `box_point_collate()` 组成 dict batch
3. （可选）`Stage1EmbedHead` 在 recycle 循环外执行一次：
   - 裁剪 buffer 原子
   - scatter 生成 `voxel_pdb_embed_grid`
   - （可选）残差加和到 `atom_feat`
   - 更新 batch 内 per-atom 字段
4. `Stage1VoxelBackbone` 输出：
   - `voxel_features`（输入通道可能因 embed head 而扩展）
   - `voxel_logits_aux`
   - `voxel_recycle_out`
5. `Stage1PointBackbone` 在命名变量生成时，通过 hook 触发：
   - voxel 特征采样
   - `concat_linear` 融合
6. `VolumePointStage1Model` 构造：
   - `atom_tokens`
   - `atom_hidden`
   - `atom_logits`
7. `VoxelPointStage1Wrapper` 计算：
   - atom 主损失
   - voxel aux 辅助损失
8. `train.py` 负责：
   - DataModule
   - DDP
   - logger
   - checkpoint
   - optimizer/scheduler 注入

当前 `forward()` 输出字典的完整键为：

- `atom_logits`
- `atom_hidden`
- `atom_tokens`
- `fused_point_feat`
- `sampled_point_fusion_feat_dict`
- `voxel_logits_aux`
- `voxel_outputs`
- `point_outputs`
- `embed_output`（舸 embed head 启用时才有值）
- `voxel_recycle_out`
- `point_recycle_out`
- `recycle_passes_used`


# 训练子系统

## 1. `VoxelPointStage1Wrapper`

当前 wrapper 位于 `src/wrappers/voxel_point_stage1.py`。

它不是待新增模块，而是当前主线训练封装。

当前职责固定为：

1. 接收 dict batch
2. 调用 `VolumePointStage1Model`
3. 计算 atom 主损失
4. 计算 voxel 辅助损失
5. 聚合 `val/atom_pr_auc`
6. 配置 optimizer/scheduler

## 2. 当前损失定义

当前 `dense.yaml` 中配置为：

- `atom_loss = BinaryFocalLossWithAlpha`
- `voxel_aux_loss = BinaryFocalLossWithAlpha`
- `atom_loss_weight = 1.0`
- `voxel_aux_loss_weight = 0.2`

当前两个损失的参数为：

- `gamma = 2.0`
- `scale = 1.0`（两者均为 1.0）
- `alpha_tune = [0.75, 0.25]`
- `use_adaptive_alpha = false`

### 2.1 atom loss

当前调用方式是：

- `outputs["atom_logits"]`
- `batch["atom_label"]`
- `hardmask=batch["atom_valid_mask"]`
- `reduction="mean"`

因此当前 atom loss 的真实监督范围由：

- `atom_valid_mask`

决定。

### 2.2 voxel aux loss

当前调用方式是：

- `outputs["voxel_logits_aux"]`
- `batch["voxel_label"]`
- `hardmask = hardmask.bool() & voxel_valid_mask.bool()`
- `reduction="mean"`

因此当前 voxel aux loss 的真实监督范围由：

1. `hardmask`
2. `voxel_valid_mask`

共同决定。

在当前 `fused` 主配置下：

- `valid_crop_margin = 0`

所以 `voxel_valid_mask` 全 True，当前实际有效范围主要由 `hardmask` 决定。


## 3. 当前验证指标

当前 wrapper 真实记录的关键指标是：

- `train/loss`
- `train/atom_loss`
- `train/voxel_aux_loss`
- `train/recycle_passes`
- `val/loss`
- `val/atom_loss`
- `val/voxel_aux_loss`
- `val/recycle_passes`
- `val/atom_pr_auc`

当前没有落地：

- `val/atom_precision`
- `val/atom_recall`



## 4. `train.py` 当前状态

`src/train.py` 当前已经实现以下行为：

### 4.1 dict sample lazy init

`_extract_input_tensor()` 已支持：

- tensor sample
- `dict["voxel_grid"]`
- tuple/list 第 0 个 tensor

因此当前 lazy init 与 dict batch 已经对齐。

### 4.2 DataLoader 与 `collate_fn`

`UnifiedDataModule._get_dataloader()` 当前逻辑为：

1. 根据 `dataloader_backend` 自动判断 PyG 或 torch DataLoader
2. 若不是 PyG，则读取 dataset 对象上的 `collate_fn`
3. 直接将该 `collate_fn` 传给 DataLoader

当前没有实现：

- 从 cfg 中显式实例化 `collate_fn`

因此文档必须写成“当前只支持 dataset 属性提供 `collate_fn`”。

### 4.3 lazy module materialization

`_initialize_lazy_modules_before_ddp()` 当前会：

1. `datamodule.setup("fit")`
2. 取 `train_ds[0]`
3. 从 sample 中提取 `voxel_grid`
4. 调用 `model.backbone.set_input_channels(in_channels)`

### 4.4 分布式策略

当前当且仅当：

- `train.devices > 1`
- 或 `train.nnodes > 1`

时启用 DDPStrategy，并固定：

- `process_group_backend="nccl"`


## 5. `default_h100` 当前训练超参

`configs/train/default_h100.yaml` 当前主配置为：

- `batch_size: 3`
- `num_workers: 20`
- `dataloader_backend: auto`
- `pin_memory: true`
- `max_epochs: 100`
- `accelerator: auto`
- `ddp_find_unused_parameters: false`
- `precision: bf16-mixed`
- `gradient_clip_val: 0.5`
- `accumulate_grad_batches: 12`
- `check_val_every_n_epoch: 1`
- `log_every_n_steps: 3`
- `optimizer: AdamW(lr=2e-5, weight_decay=0.01)`
- `scheduler: ReduceLROnPlateau(mode=max, factor=0.5, patience=10, threshold=0.001, min_lr=1e-7)`

当前这组值描述的是：

- 每卡 batch size = 3
- 每步梯度累积 = 12

因此单卡时的等效 batch 为：

- `3 * 12 = 36`

多卡或多节点时，还要再乘 world size。
