# PDB Processor Output Data Description
# PDB 预处理输出数据说明

本文件负责解析PDB, 并产生一系列.npz文件。各层次特征(原子级/残基级/图)和各个层次与ligand的结合信息都在这个文件里。但没有关于EMDB的处理。

This document describes the data structures stored in the `.npz` files generated by `Pocket\Make_Data\PDB_processor\run_preprocess.py`.
本文档说明由 `Pocket\Make_Data\PDB_processor\run_preprocess.py` 生成的 `.npz` 文件中存储的数据结构。

All files are NumPy compressed archives (`.npz`) and can be loaded using `np.load()`.
所有文件均为 NumPy 压缩归档格式（`.npz`），可使用 `np.load()` 加载。

---

## 1. atoms.npz (Atom-level Features / 原子级特征)

Stores atom coordinates, features, and metadata.
存储原子坐标、特征和元数据。

| Variable Name | Shape | Data Type | Meaning |
| :--- | :--- | :--- | :--- |
| `coords` | $(N_{atoms}, 3)$ | `float32` | Cartesian coordinates of atoms.<br>原子的笛卡尔坐标。 |
| `features` | $(N_{atoms}, 49)$ | `float32` | Atom-level feature vectors (Element + Residue Type + Physio + Mass + Density).<br>原子级特征向量（元素+残基类型+理化性质+质量+密度）。 |
| `elements` | $(N_{atoms},)$ | `str` | Element symbols (e.g., 'C', 'N').<br>元素符号（如 'C', 'N'）。 |
| `res_indices` | $(N_{atoms},)$ | `int32` | 0-based index of the residue this atom belongs to.<br>该原子所属残基的索引（从0开始）。 |
| `chain_indices` | $(N_{atoms},)$ | `int32` | 0-based index of the chain this atom belongs to.<br>该原子所属链的索引（从0开始）。 |
| `res_names` | $(N_{atoms},)$ | `str` | 3-letter residue code (e.g., 'ALA', 'G').<br>残基三字母代码（如 'ALA', 'G'）。 |
| `atom_names` | $(N_{atoms},)$ | `str` | Atom names (e.g., 'CA', 'CB').<br>原子名称（如 'CA', 'CB'）。 |

---

## 2. residues.npz (Residue-level Features / 残基级特征)

Stores residue (or nucleotide) features and local coordinate frames.
存储残基（或核苷酸）特征和局部坐标系。

| Variable Name | Shape | Data Type | Meaning |
| :--- | :--- | :--- | :--- |
| `coords` | $(N_{res}, 3)$ | `float32` | Representative atom coordinates (CA for protein, C4' for nucleotide).<br>代表原子坐标（蛋白质为 CA，核苷酸为 C4'）。 |
| `features` | $(N_{res}, 33)$ | `float32` | Residue-level feature vectors (Type One-Hot + Physio).<br>残基级特征向量（类型 One-Hot + 理化性质）。 |
| `names` | $(N_{res},)$ | `str` | Residue names (e.g., 'ALA').<br>残基名称（如 'ALA'）。 |
| `types` | $(N_{res},)$ | `str` | Residue type: 'protein' or 'nucleotide'.<br>残基类型：'protein'（蛋白质）或 'nucleotide'（核苷酸）。 |
| `chain_indices` | $(N_{res},)$ | `int32` | Chain index for each residue.<br>每个残基的链索引。 |
| `seq_numbers` | $(N_{res},)$ | `int32` | Residue sequence number in the PDB file.<br>PDB 文件中的残基序列号。 |
| `local_frames` | $(N_{res}, 3, 3)$ | `float32` | Local coordinate frame rotation matrix (column vectors are X, Y, Z axes).<br>局部坐标系旋转矩阵（列向量为 X、Y、Z 轴）。 |
| `frames_mask` | $(N_{res},)$ | `bool` | Validity mask for local frames (True if backbone atoms exist).<br>局部坐标系有效性掩码（如果骨架原子存在则为 True）。 |
| `backbone_complete` | $(N_{res},)$ | `bool` | Whether all backbone atoms are present.<br>所有骨架原子是否都存在。 |

---

## 3. graph.npz (Graph Structure / 图结构)

Stores geometric graph edges based on distance cutoff.
存储基于距离截断的几何图边。

| Variable Name | Shape | Data Type | Meaning |
| :--- | :--- | :--- | :--- |
| `edge_row` | $(N_{edges},)$ | `int32` | Source node indices (0 to $N_{atoms}-1$).<br>源节点索引（0 到 $N_{atoms}-1$）。 |
| `edge_col` | $(N_{edges},)$ | `int32` | Target node indices (0 to $N_{atoms}-1$).<br>目标节点索引（0 到 $N_{atoms}-1$）。 |
| `edge_dist` | $(N_{edges},)$ | `float32` | Euclidean distance between connected nodes.<br>连接节点之间的欧几里得距离。 |
| `edge_weight` | $(N_{edges},)$ | `float32` | Edge weight (typically $1 / (\text{distance} + \epsilon)$).<br>边权重（通常为 $1 / (\text{distance} + \epsilon)$）。 |
| `num_atoms` | `scalar` | `int` | Total number of atoms.<br>原子总数。 |
| `num_residues` | `scalar` | `int` | Total number of residues.<br>残基总数。 |
| `cutoff` | `scalar` | `float` | Distance cutoff used to build the graph (e.g., 10.0).<br>用于构建图的距离截断值（如 10.0）。 |

---

## 4. labels.npz (Labels & Ground Truth / 标签与真值)

Stores binding site labels and ligand information.
存储结合位点标签和配体信息。

| Variable Name | Shape | Data Type | Meaning |
| :--- | :--- | :--- | :--- |
| `instance_ids` | $(N_{atoms},)$ | `int32` | Binding site instance ID for each atom (-1 if background). Same as ligand ID.<br>每个原子的结合位点实例 ID（背景为 -1）。与配体 ID 相同。 |
| `ligand_ids` | $(N_{atoms},)$ | `int32` | ID of the closest ligand for each atom (even if far away).<br>每个原子最近的配体 ID（即使距离很远）。 |
| `distances` | $(N_{atoms},)$ | `float32` | Distance to the closest ligand atom.<br>到最近配体原子的距离。 |
| `binding_mask` | $(N_{atoms},)$ | `bool` | True if nearest ligand distance < threshold (4.5Å).<br>如果最近配体距离 < 阈值 (4.5Å) 则为 True。 |
| `num_ligands` | `scalar` | `int` | Number of ligands in the structure.<br>结构中的配体数量。 |
| `pocket_centers` | $(N_{ligands}, 3)$ | `float32` | Geometric centers of binding pockets.<br>结合口袋的几何中心。 |
| `ligand_resnames` | $(N_{ligands},)$ | `str` | Residue names of ligands.<br>配体的残基名称。 |
| `ligand_coords_{id}` | $(N_{lig\_atoms}, 3)$ | `float32` | Coordinates for ligand with ID `{id}` (e.g., `ligand_coords_0`).<br>ID 为 `{id}` 的配体的坐标（例如 `ligand_coords_0`）。 |
